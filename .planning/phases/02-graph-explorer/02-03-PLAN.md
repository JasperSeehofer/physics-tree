**1. Create `crates/app/src/components/graph/search.rs`** — SearchInput component:

The search component lives in the top of the graph explorer page (not in a separate nav bar for now — the existing App shell doesn't have a persistent nav. Plan 03 positions it at the top of the graph explorer layout).

```rust
use leptos::prelude::*;

/// Search input with typeahead dropdown for finding physics concepts.
/// Filters the full node list client-side (already loaded in GraphExplorerPage).
#[component]
pub fn SearchInput(
    /// Full list of nodes (titles and IDs) for client-side filtering
    #[prop(into)] nodes: Vec<(String, String, String)>, // (id, title, branch)
) -> impl IntoView
```

Implementation requirements:
- Input field: placeholder "Search concepts...", 240px width at rest, 360px on focus (CSS transition 150ms)
- Debounce: use a 200ms delay via `set_timeout` before updating filtered results (or just filter on each keystroke — with 30-50 nodes, instant filtering is fine)
- Dropdown: max 6 results, each showing concept title (14px bold) and branch label (14px, text-mist)
- Row height: 40px per result, bark-mid background, hover = leaf-green at 20% opacity
- Keyboard: ArrowUp/ArrowDown navigate results, Enter selects, Escape clears
- On selection: call `navigateToNode(node_id)` via wasm-bindgen JS bridge and update `selected_node` signal
- Empty state: "No concepts found" heading, "Try a different name or browse the graph directly" body
- The wasm-bindgen extern for `navigateToNode` is already declared in `canvas.rs`. To call it from search.rs, either:
  - Re-declare the extern in search.rs (simpler, slight duplication)
  - Move the extern declarations to a shared module (cleaner)
  - Use graph_state context to set selected_node, and let an Effect in GraphExplorerPage call navigateToNode (recommended — keeps JS calls centralized)

**Recommended approach:** SearchInput only updates the `selected_node` signal. The GraphExplorerPage has an Effect that watches `selected_node` and calls `navigateToNode` + fetches prereqs. This keeps all JS bridge calls in one place.
