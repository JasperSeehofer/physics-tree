| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Streak counter gap to icon, mastery badge margin |

## Component Inventory

New and modified components required for this phase:

### Dashboard: Stats Cards (Modified)

**`StatsCards` — streak card wire-up**
- Replace em-dash streak placeholder with live integer from new streak API.
- Streak icon: existing flame SVG in `text-sky-teal` (unchanged).
- Active streak value: `text-3xl font-bold text-petal-white` — e.g. "14".
- Streak-broken state (streak = 0 after previous streak > 0): value shows "0" in `text-bloom-pink` with a small "Streak lost" sub-label in `text-xs text-bloom-pink`.
- Freeze token indicator: small badge below the streak value — `text-xs text-mist` — "1 freeze token available" or "No freeze tokens".

### Dashboard: MiniTree (Upgraded)

**`MiniTree` — botanical growth stage upgrade from static circles**
- Replace circle SVG elements with tier-aware growth-stage SVG shapes per D-14.
- Growth stage shapes (inline SVG paths, no external dependencies):
  - **Seed / dormant** (`mastery_level = 0`): filled circle r=4, `fill="var(--color-bark-light)"` — small, dim dot.
  - **Bud / sprout** (bronze, mastery_level 1–149): circle r=6 with 3 upward petal-stub paths, `fill="var(--color-sun-amber)"` at 80% opacity.
  - **Open leaf** (silver, mastery_level 150–299): circle r=7 with 4-point leaf outline, `fill="var(--color-mist)"`.
  - **Flowering / bloom** (gold, mastery_level 300+): circle r=8 with 6-petal flower outline, `fill="var(--color-leaf-green)"` full opacity.
- CSS entrance animations on first load (Tailwind `animate-` utilities or inline SVG SMIL):
  - Bloom nodes: `scale-0` → `scale-100` over 400ms with `ease-out` timing, staggered 50ms per node.
  - Bud nodes: fade-in `opacity-0` → `opacity-100` over 300ms.
  - Seed nodes: no animation (already rendered, dormant).
- Connection lines between tiers: unchanged from Phase 4 pattern (`stroke="var(--color-bark-mid)" stroke-width="2"`).
- Node hover: existing `cursor-pointer` + title tooltip; add mastery tier label to tooltip text — "{title} — {tier}" where tier is "Seed", "Bud", "Leaf", or "Bloom".
- Empty state: preserve existing copy from Phase 4 (unchanged).

### Dashboard: Streak Detail Row (New)

**`StreakDetail` component — below StatsCards row**
- Container: `bg-bark-dark rounded-card p-4 flex items-center justify-between`
- Left: flame icon `text-sky-teal w-5 h-5` + streak count `text-xl font-bold text-petal-white` + label `text-sm text-mist ml-2` — "day streak"
- Right: freeze token badge `bg-bark-mid rounded px-3 py-1 text-sm text-mist flex items-center gap-1` — snowflake inline SVG icon `text-sky-teal w-4 h-4` + "{N} freeze" text
- Streak milestone banner (shown on streak milestone days — 7/14/30/etc.): `bg-sun-amber/10 border border-sun-amber/30 rounded-card p-4 text-sm text-sun-amber` — "You earned a freeze token! Day {N} streak milestone."
- Hidden when streak = 0 and no tokens held.

### Graph Explorer: Botanical Node Program (New JS)

**`botanicalNodeProgram` in `sigma_bridge.js` — custom Sigma.js canvas node renderer**
- Registered as a custom Sigma node type; replaces `circle` for nodes where the authenticated user has progress data.
- Growth stage mapped from mastery tier at render time:
  - Seed: filled circle r=6, color `#2e3340` (bark-light token).
  - Sprout/bronze: filled circle r=7 + 3 small upward petal paths at top, color `#f4b942` (sun-amber).
  - Leaf/silver: filled rounded diamond r=8, color `#8892a4` (mist).
  - Bloom/gold: filled circle r=9 + 6-point star/flower outline in `#4caf7d` (leaf-green), glow via `ctx.shadowColor = "#4caf7d"; ctx.shadowBlur = 8`.
- Unowned nodes (not in user's progress set) continue to use existing `botanicalNodeReducer` depth-tier styles (no change).
- Progressive reveal (D-13): nodes outside the learned frontier are hidden via `hidden: true` in the Sigma node attributes — this uses the existing `botanicalNodeReducer hidden=true` pattern (established Phase 2 pattern).
- Integration: `initSigma()` accepts an optional `userProgress` map `{nodeId: masteryTier}` passed from WASM via wasm-bindgen. When present, the custom node program takes precedence over depth-tier styling for that node.

### Quiz: XP Award Feedback (New)

**XP toast on quiz checkpoint pass**
- Trigger: server response confirms XP awarded after quiz submission scoring ≥ 70%.
- Toast container: `fixed bottom-6 right-6 z-50 bg-bark-dark border border-leaf-green rounded-card px-4 py-3 flex items-center gap-3 shadow-lg`
- XP icon: inline SVG star `text-sun-amber w-5 h-5`
- Text: `text-base font-bold text-petal-white` — "+{XP} XP" + `text-sm text-mist ml-1` — concept name
- Perfect score suffix: `text-sm text-sun-amber` — "1.5× perfect score bonus!" (only if 100% score)
- Auto-dismiss after 4 seconds. Fade-out `opacity-100` → `opacity-0` over 300ms.
- Streak milestone notification stacks below XP toast if earned on the same action: same container style, `text-sky-teal` icon, "Day {N} streak!" text.

### Concept Page: Mastery Badge (New)

**Mastery tier badge on ConceptPage header**
- Shown when user is authenticated and has any progress on this concept.
- Badge: `inline-flex items-center gap-1 rounded px-2 py-0.5 text-xs font-bold` — color per tier:
  - Bronze: `bg-sun-amber/20 text-sun-amber` — "Bronze"
  - Silver: `bg-mist/20 text-mist` — "Silver"
  - Gold: `bg-leaf-green/20 text-leaf-green` — "Gold"
- Positioned: after the concept title heading, on the same line on desktop (`flex items-baseline gap-3`), below on mobile.
- XP sub-label: `text-xs text-mist` — "{N} / {threshold} XP to next tier" (omitted at Gold).
- Hidden for unauthenticated users (no mastery state exists).

## Interaction Contracts

### Quiz XP Award Flow
- After quiz submission response returns with XP > 0: show XP toast immediately (no page reload).
- If quiz response includes a streak milestone: stack streak notification below XP toast.
- If quiz response includes a freeze token earned: stack freeze token notification as third item.
- Toast z-index: 50 (above content, below modals if any).
- Toast keyboard: not focusable (purely informational). Screen reader: `role="status" aria-live="polite"`.

### Streak Display
- Streak value on dashboard stats card: fetched via same dashboard API call (no separate request).
- Freeze token count: fetched in same call.
- Streak milestone banner: shown only on page load if the current session's qualifying action earned a new token. Dismissed on click or after 8 seconds. No auto-dismiss fade — full opacity until dismissed.
- `aria-live="polite"` on the streak region so screen reader announces updates.

### Botanical MiniTree Animations
- Entrance animation on component mount only (not on every re-render).
- Animation disabled if `prefers-reduced-motion: reduce` media query is active. Use `@media (prefers-reduced-motion: reduce)` to set `transition: none` and skip SMIL animations.
- Animation stagger: bloom nodes animate first (highest priority), then leaf, then bud, then seed (no animation). Stagger delay = node index × 50ms within each tier group.

### Graph: Progressive Reveal Interaction
- On initial graph load for authenticated users: fetch user progress map from API. Until response arrives, render all nodes with existing depth-tier styling (no flash of empty state).
- After progress map loaded: re-apply node attributes via `sigmaInstance.refresh()`. Nodes outside the frontier become `hidden: true` with a 200ms CSS fade-out on the canvas overlay.
- "Reveal" effect: frontier nodes (immediate prerequisites and unlocks of learned nodes) are shown at seed stage (dim) to communicate discoverability without confusion.
- Nodes hidden by progressive reveal are not keyboard-focusable and not in the search index (exclude from `filter_nodes` results for nodes with `hidden: true`).

### Mastery Badge on Concept Page
- Shown on page load if user is authenticated; fetched as part of the existing concept page API or a lightweight `/api/progress/{slug}` endpoint.
- Badge does not block page render — shown only after auth check resolves.
- Clicking the badge: no action (informational only). `aria-label="{tier} mastery for this concept"`.

### Responsive Breakpoints
- Unchanged from Phase 4: `sm:` = 640px, `md:` = 768px, `lg:` = 1024px.
- XP toast: `fixed bottom-6 right-6` on desktop; `fixed bottom-4 left-4 right-4` on mobile (full-width, centered).
- Streak detail row: hidden on mobile (xs/sm) — streak data visible only in the stats card grid. Shown at `md:` and above.

## Accessibility Contract

- XP toast: `role="status" aria-live="polite"` — announced by screen readers without interrupting flow.
- Streak milestone banner: `role="alert" aria-live="assertive"` — interrupts screen reader if newly earned.
- Mastery badges: `aria-label="{tier} mastery"` on the badge element; do not rely on color alone — text tier name always rendered.
- Growth stage nodes in MiniTree SVG: tooltip `<title>` element updated to include tier — "{title} — {tier}". Overall SVG `aria-label` unchanged.
- Botanical node renderer in Sigma.js: accessible label unchanged from Phase 2/3 patterns (title attribute on hover tooltip). Growth stage is communicated via the tooltip text, not color alone.
- Freeze token indicator: `aria-label="{N} freeze tokens available"` on the badge container.
- Animation: all CSS transitions and SMIL animations respect `prefers-reduced-motion: reduce`.

## XP and Mastery Thresholds (Claude's Discretion — Reference Values)

These are implementation reference values for the backend. Included here so the executor has a single source of truth.

| Tier | Cumulative Concept XP Threshold |
|------|--------------------------------|
| None (seed) | 0–49 XP |
| Bronze (sprout) | 50–149 XP |
| Silver (leaf) | 150–299 XP |
| Gold (bloom) | 300+ XP |

XP base amounts per concept depth tier:
- Root concepts: 15 XP base
- Trunk concepts: 20 XP base
- Branch concepts: 30 XP base
- Leaf concepts: 40 XP base

XP multipliers:
- Score proportion multiplier: `score_pct / 100.0` (range 0.70–1.0 for passing quizzes; clamped at 70% minimum)
- Perfect score bonus: 1.5× (applied after score proportion, so max = base × 1.5)
- Example: branch concept, 85% score = 30 × 0.85 = 25.5 → rounded to 26 XP

Streak milestone schedule (when freeze tokens are earned):
- Day 7, 14, 30, 60, 90, then every 90 days thereafter
- Maximum freeze token capacity: 3 tokens
