## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline element nudges |
| sm | 8px | Compact input padding, tag spacing, avatar gap to username |
| md | 16px | Form field internal padding (`px-4 py-2`), card content padding |
| lg | 24px | Card-to-card gap, section spacing within dashboard |
| xl | 32px | Stats card row gap, section top/bottom padding |
| 2xl | 48px | Dashboard section breaks, auth page hero-to-form gap |
| 3xl | 64px | Page vertical padding (`py-16` on auth page wrappers) |

Exceptions:
- Touch targets for all interactive elements minimum 44px height (navbar avatar button, hamburger toggle, stats card links). Use `min-h-[44px]` utility.
- Navbar height: 56px (`h-14`) — matches established Phase 1 shell height.

## Typography

Exactly 4 sizes, 2 weights. Source: existing codebase patterns from phases 1–3 (`crates/app/src/**/*.rs`).

| Role | Size | Tailwind Class | Weight | Weight Class | Line Height |
|------|------|----------------|--------|--------------|-------------|
| Label / caption | 14px | `text-sm` | 400 | `font-normal` | 1.4 (`leading-snug`) |
| Body | 16px | `text-base` | 400 | `font-normal` | 1.6 (`leading-relaxed`) |
| Heading | 20px | `text-xl` | 700 | `font-bold` | 1.2 (`leading-tight`) |
| Display | 28px | `text-3xl` | 700 | `font-bold` | 1.15 (`leading-tight`) |

Notes:
- `font-bold` maps to Nunito weight 700.
- `font-extrabold` (800) is reserved for the wordmark logotype only — do not introduce in Phase 4 components.
- Stats card numbers (XP total, concept count, streak) use `text-3xl font-bold` to feel like display values.
- Form labels use `text-sm font-bold` (14px/700) — label above input, not placeholder-as-label.
- Error and hint text under form fields use `text-sm font-normal text-bloom-pink` (errors) or `text-sm font-normal text-mist` (hints).

## Color

Source: `style/main.css` `@theme` block — all tokens already exist.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--color-void` | `#0d0f14` | Page background, auth page full-screen backdrop |
| Secondary (30%) | `--color-bark-dark` / `--color-bark-mid` | `#1a1d24` / `#252932` | Form cards, stats cards, navbar background, avatar dropdown panel |
| Accent (10%) | `--color-leaf-green` | `#4caf7d` | See reserved-for list below |
| Destructive | `--color-bloom-pink` | `#e8547a` | Log Out button (text only, no fill), inline form errors |

Accent `--color-leaf-green` reserved for:
1. Primary CTA buttons only ("Create Account" button fill, "Log In" button fill)
2. Active nav link indicator (underline or left border)
3. Focus ring on form inputs (`focus:border-leaf-green` — established pattern from `search.rs`)
4. Mini tree node fill for mastered concepts (colored by progress intensity)
5. Logged-in state avatar ring (`ring-2 ring-leaf-green`)

Secondary semantic colors (non-destructive):
- `--color-sun-amber` (#f4b942): "Verify your email" banner background tint (`bg-sun-amber/10 text-sun-amber`) — D-03 optional verification banner.
- `--color-sky-teal` (#3fc8d4): Stats card icons for "streak" metric (consistent with its use as a highlight accent in content pages).
- `--color-mist` (#8892a4): Secondary text, disabled states, placeholder text, "0" placeholder values in Phase 4 dashboard cards awaiting Phase 5 logic.

## Interaction Contracts

### Form Validation
- Validate on blur (field loses focus), not on keystroke.
- Show inline error below the relevant field immediately after blur if invalid.
- On submit with errors: re-validate all fields, scroll to first error, do not clear values.
- On submit success (login): redirect to `/dashboard`.
- On submit success (register): redirect to `/dashboard` with welcome state (no separate welcome page).
- Loading state during submit: button text becomes "Logging in…" / "Creating account…" and is `disabled` with `opacity-60`.

### Avatar Dropdown
- Trigger: click (not hover). Toggle open/close.
- Close on: outside click, Escape key, navigation to new route.
- Keyboard: focus-trapped within open dropdown, arrow-key navigation between items.

### Hamburger Menu (mobile)
- Trigger: click. Slides down, pushing page content down (not overlay).
- Close on: tap a nav link, tap outside, Escape key.
- `aria-expanded` on trigger button reflects open/closed state.

### Mini Tree Nodes
- Hover: show tooltip with concept title and branch. Reuse tooltip component pattern from `tooltip.rs`.
- Click: navigate to `/learn/{slug}` (standard `leptos_router` `<A>` element).
- No hover state on unlearned nodes that have no progress (tooltip still shows title for discoverability).

### Responsive Breakpoints
- `sm:` prefix = 640px (phone landscape — minimum supported width per D-17)
- `md:` prefix = 768px (tablet — hamburger menu collapses, bottom sheet activates)
- `lg:` prefix = 1024px (desktop — full layout with sidebars)

## Accessibility Contract

- All form inputs: `<label>` elements with explicit `for` attribute (not placeholder-as-label).
- All buttons: descriptive text or `aria-label` for icon-only buttons (hamburger: `aria-label="Open navigation menu"`).
- Avatar button: `aria-label="Account menu"` when logged in, `aria-label="Log in"` when guest.
- Error messages: `role="alert"` for server-side errors (invalid credentials). Inline field errors linked via `aria-describedby`.
- Mini tree SVG: `role="img"` with `aria-label="Knowledge tree showing your learning progress"`. Individual clickable nodes use `<a>` with `aria-label="{title} — open concept"`.
- Color is never the sole indicator of state — mastery levels in mini tree use both color and opacity difference.
- Focus ring: `focus-visible:ring-2 focus-visible:ring-leaf-green focus-visible:outline-none` on all interactive elements.
