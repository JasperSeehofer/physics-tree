**Step 2: Create `crates/server/src/bin/` directory and `validate.rs`:**

The binary takes a single argument: path to a node directory (e.g., `content/classical-mechanics/kinematics`).

Implementation flow:
1. Parse CLI arg: `let dir = std::env::args().nth(1).expect("Usage: validate <node_dir>");`
2. Check `--json` flag: `let json_output = std::env::args().any(|a| a == "--json");`
3. Read `{dir}/node.yaml` using `std::fs::read_to_string`. On read error, print `node.yaml: file not found` and exit 1.
4. Parse YAML with serde-saphyr: `let meta: NodeMeta = serde_saphyr::from_str(&yaml_str)`. On parse error, print the serde error with `node.yaml:parse  {error}` format and exit 1.
5. For each phase number 0-6:
   a. Check if `{dir}/phase-{n}.md` exists — record in `phase_files_found: Vec<u8>`
   b. If file exists, read it, use `gray_matter::Matter::<gray_matter::engine::YAML>::new().parse(&content)` to split frontmatter from body
   c. Use `domain::content_spec::extract_h2_headings(&body)` to get H2 headings from the Markdown body
   d. Store in `phase_headings: HashMap<u8, Vec<String>>`
6. Build `ParsedNode { meta, phase_files_found, phase_headings }`
7. Call `domain::content_spec::validate_node(&parsed_node)`
8. If errors is empty: print `OK: {dir} is valid` to stdout and exit 0
9. If errors is non-empty:
   - If `--json` flag: print `serde_json::to_string_pretty(&errors)` to stdout
   - Otherwise: print each error using its Display impl (one per line) to stderr
   - Exit with code 1

Usage examples (document in a comment at top of file):
```
// cargo run --bin validate --features ssr -- content/classical-mechanics/kinematics
// cargo run --bin validate --features ssr -- content/classical-mechanics/kinematics --json
```
