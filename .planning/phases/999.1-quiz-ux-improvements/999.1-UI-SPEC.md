## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`, `gap-0.5`) |
| sm | 8px | Compact element spacing (`space-y-2`, `gap-2`) |
| md | 16px | Default element spacing (`px-4`, `space-y-4`) |
| lg | 24px | Section padding (`bottom-6`, `right-6`) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Touch targets for "Check answer" / "Check formula" CTA buttons use `px-4 py-2` (16px √ó 8px), which satisfies WCAG 44√ó44px minimum when combined with font size. No additional exceptions for this phase.

Checkmark icon: Unicode `U+2713` (‚úì) in a `<span class="mr-2 font-bold">` sibling to the option text `<span>`, rendered only when `QuizState::Correct && is_correct`. Per CONTEXT.md D-03.

Transition: `transition-colors duration-200` on the button element (existing class already present).

Source: CONTEXT.md D-01, D-02, D-03; RESEARCH.md Pattern 1 + Pitfall 2.

### Formula Input Area (Correct State)

When `FormulaState::Correct`:
- Input field: `bg-leaf-green border-leaf-green text-void` (solid green fill, dark text)
- Input field disabled
- Preview box below: `bg-leaf-green/20 border-leaf-green` (tinted green to echo correct state)
- Feedback text: `text-leaf-green text-sm font-bold` "Correct!" (existing pattern, unchanged)

Source: CONTEXT.md D-01 (formula input area included in all 3 question types).

### Matching Pair Cells (Correct State)

When `MatchState::Checked { correct: true }`:
- Both left and right cells for all pairs: `bg-leaf-green text-void` solid fill
- Checkmark icon (`U+2713`) prepended to each pair cell text

When `MatchState::Checked { correct: false }` (hint shown):
- Incorrect pairs: existing styling (no change specified)
- `hint_shown` signal set to `true` to track hint usage

Source: CONTEXT.md D-01 ("all matching pairs"); RESEARCH.md Pitfall 6.

### LaTeX Rendering in Quiz Text

All four quiz string fields ‚Äî question text, hint text, explanation text, and MC option labels ‚Äî are pre-processed server-side to replace `$...$` and `$$...$$` with `<span data-latex="..." data-display="false"></span>` / `<div data-latex="..." data-display="true"></div>` placeholders.

These fields are rendered via Leptos `inner_html` directive on a child `<span>` (for inline text) or `<div>` (for block-level).

`renderAllPlaceholders()` is called from a `cfg(target_arch = "wasm32")` `Effect::new` after component mount, following the same pattern as `concept.rs` lines 284‚Äì289.

Source: CONTEXT.md D-04, D-05, D-06; RESEARCH.md Patterns 2 and 3.

### Formula Input Live Preview

The preview `<div>` below the formula input renders KaTeX HTML via `inner_html`. The fix investigates brace/underscore stripping in the `event_target_value ‚Üí katex_render ‚Üí JsValue::from_str` data path (CONTEXT.md D-08, D-09). Visual appearance of the preview box is unchanged: `px-3 py-2 bg-bark-mid rounded border border-bark-light min-h-[2.5rem] flex items-center`.

### XP Toast ‚Äî Hint Penalty Line

When `XpAwardData.hint_penalty == true`, render an additional line inside the toast body:

```
<span class="text-sm text-sky-teal">"hint penalty applied"</span>
```

Placed below the XP amount line, above the `perfect_bonus` line. Uses `sky-teal` to match the informational (non-punitive) tone of the existing freeze-used notification.

Full toast copy when hint penalty active: `"+{N} XP"` on the primary line, `"hint penalty applied"` as secondary line in `text-sky-teal`.

Source: CONTEXT.md D-12; RESEARCH.md Pattern 5 Layer 6. `sky-teal` chosen to match the educational/neutral tone of freeze-used copy ‚Äî not `bloom-pink` which signals failure.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA ‚Äî MC | "Check answer" |
| Primary CTA ‚Äî formula | "Check formula" |
| Correct feedback | "Correct!" |
| Hint feedback | "Not quite ‚Äî {hint text} Try again." |
| Revealed feedback ‚Äî MC | "The answer is shown above. Here's why:" |
| Revealed feedback ‚Äî formula | "The expected formula is: {formula} {explanation}" |
| XP toast ‚Äî standard | "+{N} XP  {concept name}" |
| XP toast ‚Äî perfect bonus | "1.5x perfect score bonus!" (sun-amber, below XP line) |
| XP toast ‚Äî hint penalty | "hint penalty applied" (sky-teal, below XP line) |
| XP toast ‚Äî freeze used | "Freeze token used. Streak protected for today." (sky-teal) |
| Empty state ‚Äî no quiz | Not applicable (quiz is always loaded with concept content) |
| Error state ‚Äî formula check fails | Silent no-op (existing behavior: button stays active, user retries) |

Destructive actions in this phase: none. No deletions or irreversible actions.

Notes on copy tone:
- Hint penalty message is "hint penalty applied" ‚Äî factual, not punitive. Not "XP reduced for cheating." Per CONTEXT.md Specifics.
- "Correct!" short and celebratory ‚Äî matching the bold Kurzgesagt aesthetic.

Source: CONTEXT.md D-12 and Specifics; existing `multiple_choice.rs` and `formula_input.rs` copy preserved.

## Accessibility Notes

- All interactive buttons retain `disabled` attribute when locked (existing pattern).
- XP toast retains `role="status"` and `aria-live="polite"` (existing).
- Checkmark icons are decorative (`aria-hidden="true"` not needed for Unicode inline spans, but can be added).
- Color alone does not convey correct/incorrect state ‚Äî the checkmark icon provides a non-color indicator for correct answers (WCAG 1.4.1). Wrong answers retain border-only treatment (no checkmark needed per D-02 ‚Äî wrong answers are obvious from the absence of green fill).
- `prefers-reduced-motion` already handled in `style/main.css` for `animate-fade-in` and `animate-scale-in`. The `transition-colors duration-200` on quiz buttons is low-motion-impact and does not require suppression.
