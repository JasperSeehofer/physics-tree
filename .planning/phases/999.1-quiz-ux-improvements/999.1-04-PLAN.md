Two root causes to fix:

**Part A: KaTeX CSS not loaded**

The `import 'katex/dist/katex.min.css'` in `katex_bridge.js` (line 2) is silently dropped by esbuild because no CSS extraction is configured. The KaTeX HTML structure renders correctly but without styling, fractions appear as flat text.

Fix: Add a `<link>` tag to load KaTeX CSS from the npm package directly (or from a CDN). In `crates/app/src/lib.rs`, in the `shell()` function's `<head>` section, add BEFORE the katex_bundle.js script tag:

```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/katex@0.16.40/dist/katex.min.css" crossorigin="anonymous" />
```

Use the exact version from package.json (katex ^0.16.40 -> 0.16.40). The `crossorigin="anonymous"` is required for CORS font loading.

Also in `katex_bridge.js`, remove the `import 'katex/dist/katex.min.css';` line (line 2) since CSS is now loaded via link tag. Keep the `import katex from 'katex';` line.

**Part B: Effect timing — renderAllPlaceholders fires before DOM update**

In all 3 quiz components (multiple_choice.rs, formula_input.rs, matching.rs), the Effect that calls `renderAllPlaceholders()` runs synchronously during Leptos reactive graph evaluation — BEFORE the browser commits `inner_html` content to the DOM. When `document.querySelectorAll('[data-latex]')` runs, those elements don't exist yet.

Fix: In each quiz component's Effect, wrap the renderAllPlaceholders call in `requestAnimationFrame` so it executes after the browser's next paint (when DOM has been updated):

Replace the direct `func.call0(&bridge)` call with:

```rust
#[cfg(target_arch = "wasm32")]
Effect::new(move |_| {
    let _ = state.get(); // subscribe to state changes (or match_state for matching.rs)
    use wasm_bindgen::JsValue;
    use wasm_bindgen::closure::Closure;
    let window = web_sys::window().unwrap();
    // Defer to next animation frame so DOM has committed inner_html
    let cb = Closure::<dyn FnMut()>::new(move || {
        let window = web_sys::window().unwrap();
        if let Ok(bridge) = js_sys::Reflect::get(&window, &JsValue::from_str("__katex_bridge")) {
            if let Ok(func) = js_sys::Reflect::get(&bridge, &JsValue::from_str("renderAllPlaceholders")) {
                let func: js_sys::Function = func.into();
                let _ = func.call0(&bridge);
            }
        }
    });
    let _ = window.request_animation_frame(cb.as_ref().unchecked_ref());
    cb.forget(); // Leak is bounded — one per state change, not per frame
});
```

Note: `unchecked_ref()` comes from `wasm_bindgen::JsCast` — add `use wasm_bindgen::JsCast;` inside the cfg block.

Apply the same fix to the concept page content hydration Effect in `concept.rs` (lines 274-307). The renderAllPlaceholders call there (lines 301-307) should also be wrapped in requestAnimationFrame. Keep the other hydration calls (misconception cards, derivation steps, concept links, TOC scroll spy) inside the same rAF callback since they also need DOM to be committed.

Important: The `Closure::forget()` here is acceptable — each rAF callback fires once and is garbage collected by the browser after execution. The closure memory is tiny and bounded by the number of state transitions.
