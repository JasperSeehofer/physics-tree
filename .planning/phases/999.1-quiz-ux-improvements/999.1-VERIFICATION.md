### Human Verification Required

#### 1. LaTeX Renders in Quiz Question Text

**Test:** Navigate to a concept page (e.g., `/concepts/kinematic-equations`). Open a multiple choice quiz checkpoint. Inspect the question text for math expressions like $x(t) = 3t^2$ or $\frac{1}{2}mv^2$.
**Expected:** Superscripts render as raised characters, fractions render as stacked fractions, Greek symbols render as proper symbols — no raw dollar-sign text and no unstyled flat KaTeX HTML.
**Why human:** Root causes are fixed in code (CDN CSS link tag, rAF deferral), but correct styling requires the browser to load the CDN stylesheet and execute KaTeX.

#### 2. Formula Input Preview Renders Structured LaTeX

**Test:** Navigate to a concept page with a formula-type quiz question. Type `\frac{1}{2}` into the formula input field.
**Expected:** The live preview below the input renders a proper mathematical fraction (1 over 2, stacked), not flat text "21" and not raw LaTeX text.
**Why human:** Both fixes verified in code (CSS CDN link, rAF deferral, `prop:value` removed), but actual KaTeX rendering through wasm-bindgen + browser requires live testing.

#### 3. XP Toast Shows Hint Penalty (Authenticated)

**Test:** Log in to an account. Navigate to a concept page with quiz checkpoints. On a multiple choice question, select the wrong answer (hint appears), then select the correct answer. Complete remaining checkpoints. Observe the XP toast.
**Expected:** XP toast appears with a reduced XP amount and sky-teal text "hint penalty applied" below the XP number.
**Why human:** The full data flow is wired at every link but requires a live authenticated session and browser interaction.

#### 4. Login Nudge Appears When Unauthenticated

**Test:** Without logging in (clear session cookies if needed), navigate to a concept page. Complete all quiz checkpoints correctly.
**Expected:** After completing all checkpoints, an inline info bar appears below the quiz area showing a sky-teal info icon and the text "Log in to track your progress and earn XP." with "Log in" as a link to `/login`.
**Why human:** `login_nudge` signal and conditional render are verified in code, but the browser needs to make the actual POST request, receive the 401, and trigger the signal update.

### Gaps Summary

No gaps found. All 16 truths (13 original + 3 new from gap closures in plans 04-05) are implemented with substantive code and correct wiring. Four items require human browser verification for visual confirmation — these are rendering behaviors that cannot be tested without a live browser.

**Gap closure summary (plans 04 and 05):**
- Plan 04 fixed the two root causes behind UAT failures 4 and 5: KaTeX CSS was not loading (esbuild silently drops CSS imports — fixed by CDN link tag in HTML shell) and `renderAllPlaceholders` was firing before DOM committed `inner_html` (fixed by `requestAnimationFrame` deferral in all 3 quiz components and concept page).
- Plan 05 fixed UAT failure 6: `post_award_xp` silently swallowed 401 errors. Now returns `Result<AwardXpResponse, bool>` with explicit 401 detection, and unauthenticated users see a login nudge instead of nothing.

**Notable observation:** `review.rs` discards `hint_used: bool` from quiz components (lines 268, 280, 292 use `_hint_used`) — hint penalty is not applied to spaced repetition reviews. This is outside phase scope but represents a future consistency gap.
