# Phase 999.1 Plan 04: KaTeX CSS Loading and Effect Timing Fix Summary

**One-liner:** KaTeX CSS now loads via CDN link tag and renderAllPlaceholders deferred via requestAnimationFrame to fix quiz LaTeX rendering.

## What Was Built

### Root Cause A: KaTeX CSS not loaded in browser

`katex_bridge.js` had `import 'katex/dist/katex.min.css'` which esbuild silently drops when no CSS extraction is configured. KaTeX HTML was generated with the correct structure but rendered unstyled — fractions appeared as flat text ("21" instead of a stacked fraction).

**Fix:** Added `<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/katex@0.16.40/dist/katex.min.css" crossorigin="anonymous" />` to the HTML shell head in `lib.rs`, and removed the CSS import from `katex_bridge.js`. Rebuilt `public/js/katex_bundle.js`.

### Root Cause B: renderAllPlaceholders fires before DOM update

Leptos `Effect::new` callbacks execute synchronously during reactive graph evaluation — before the browser commits `inner_html` content to the DOM. When `renderAllPlaceholders` called `document.querySelectorAll('[data-latex]')`, those elements did not exist yet.

**Fix:** Wrapped `renderAllPlaceholders` calls in `requestAnimationFrame` closures in all 3 quiz components (multiple_choice, formula_input, matching) and in the concept page content hydration Effect. The rAF callback fires after the browser's next paint when DOM has been updated.

## Key Decisions

1. **KaTeX CSS via CDN link tag:** The plan specified using `cdn.jsdelivr.net/npm/katex@0.16.40/dist/katex.min.css` with `crossorigin="anonymous"`. This avoids esbuild CSS extraction configuration entirely and loads CSS reliably.

2. **requestAnimationFrame deferral:** All DOM hydration (KaTeX rendering, misconception cards, derivation steppers, concept links, TOC scroll-spy) moved inside the rAF callback in concept.rs to ensure all DOM-dependent operations run after DOM update, not just KaTeX.

3. **Closure::forget() for rAF callbacks:** Each rAF callback fires once and is garbage collected by the browser after execution. The leak is bounded by the number of state transitions (not per-frame), making it acceptable.
