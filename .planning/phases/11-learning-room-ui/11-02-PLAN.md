**Step 1: Fenced div pre-pass** (before pulldown-cmark parsing). The existing function already has a regex pre-pass for `::directive[name]{attrs}`. Extend it to handle fenced divs:

- `:::definition\n...\n:::` -> `<div class="definition-block">\n...\n</div>`
- `:::collapse TITLE\n...\n:::` -> `<details><summary>TITLE</summary>\n...\n</details>`
- `:::figure\n...\n:::` -> `<figure class="figure-block">\n...\n<figcaption>LAST_LINE</figcaption></figure>` (last non-empty line before closing `:::` becomes figcaption)

Use a regex like `r"(?m)^:::(definition|collapse|figure)\s*(.*)\n([\s\S]*?)\n:::\s*$"` to capture the directive type, optional title, and content.

**Step 3: Custom event consumer.** Replace `pulldown_cmark::html::push_html` with a manual `for event in parser` loop. Handle:

- `Event::InlineMath(latex)` -> `<span data-latex="{escaped}" data-display="false"></span>`. HTML-attribute-escape the latex string.
- `Event::DisplayMath(latex)` -> `<div data-latex="{escaped}" data-display="true"></div>`. HTML-attribute-escape.
- `Event::Start(Tag::BlockQuote(Some(kind)))` -> open admonition div. Map `BlockQuoteKind::Note` -> `("admonition-note", "Note")`, `Tip` -> `("admonition-tip", "Tip")`, `Important` -> `("admonition-important", "Important")`, `Warning` -> `("admonition-warning", "Warning")`, `Caution` -> `("admonition-caution", "Caution")`. Emit `<div class="admonition {class}"><span class="admonition-label">{label}</span>`.
- `Event::End(TagEnd::BlockQuote)` -> `</div>` (only when inside an admonition, otherwise `</blockquote>`). Track admonition state with a boolean flag `in_admonition`.
- `Event::Start(Tag::CodeBlock(CodeBlockKind::Fenced(lang)))` when `lang == "quiz"` -> start buffering quiz YAML. Set `in_quiz_block = true`.
- `Event::Start(Tag::CodeBlock(CodeBlockKind::Fenced(lang)))` when `lang != "quiz"` -> start buffering code text. Set `in_code_block = true`, store `lang`.
- `Event::Text(text)` when `in_quiz_block` -> append to quiz buffer.
- `Event::End(TagEnd::CodeBlock)` when `in_quiz_block` -> emit `<div data-quiz-block="{html_attribute_escaped_yaml}"></div>`. Reset `in_quiz_block`.
- `Event::Text(text)` when `in_code_block` -> append to code buffer.
- `Event::End(TagEnd::CodeBlock)` -> run syntect highlighting on buffered code with the stored lang. Emit `<pre class="highlight"><code>{highlighted_html}</code></pre>`. Use `#[cfg(feature = "ssr")]` for the syntect path; provide a plain `<pre><code>{escaped}</code></pre>` fallback for non-ssr (though renderer is ssr-only).
- `Event::Start(Tag::Heading { level, id, .. })` -> emit `<h{level} id="{id_or_slugified}">`. If `id` is `Some`, use it; otherwise slugify the heading text. Track headings to populate `sections` vec for h2s.
- All other events -> use `pulldown_cmark::html::push_html(&mut buf, std::iter::once(event))` as fallback, then append `buf` to `html`.

**Step 4: syntect initialization.** Use `std::sync::OnceLock` for `SyntaxSet` and `ThemeSet` singletons:

#[cfg(feature = "ssr")]

Use `SyntaxSet::load_defaults_newlines()` and `ThemeSet::load_defaults()`. Theme: `"base16-ocean.dark"` (dark theme matching the Kurzgesagt void aesthetic).
