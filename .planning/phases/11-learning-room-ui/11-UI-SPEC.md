## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline label spacing |
| sm | 8px | Compact element spacing, tab inner padding, pill padding |
| md | 16px | Default element spacing, card padding, breadcrumb gaps |
| lg | 24px | Section padding, tab bar vertical padding, phase header padding |
| xl | 32px | Layout gaps between major content blocks |
| 2xl | 48px | Phase section breaks, vertical breathing room in content area |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Touch targets for tab items: minimum 44px height on mobile (accessibility requirement)
- Progress bar height: 4px (sub-scale, purely decorative)
- Phase color accent sidebar stripe: 4px width (sub-scale decorative element)

Source: standard 8-point scale; exceptions follow WCAG 2.5.5 minimum touch target guidance.

## Typography

Primary focal point: node title (Display/28px) draws the eye on entry; tab bar is the primary navigation anchor.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 700 (bold) | 1.4 |
| Heading | 20px | 700 (bold) | 1.2 |
| Display | 28px | 700 (bold) | 1.2 |

Notes:
- All sizes use Tailwind utility classes mapped to this scale: `text-sm` (14px), `text-base` (16px), `text-xl` (20px), `text-[28px]` (28px)
- Font weights map to: `font-normal` (400), `font-bold` (700)
- Body text uses `text-petal-white` (`#f0f2f5`) on dark surfaces
- Secondary/meta text uses `text-mist` (`#8892a4`)
- Tab labels are 14px / 700 weight

Source: `concept.rs` uses `text-[28px] font-bold leading-[1.2]` for h1, `text-base` for body, `text-sm` for meta — confirmed as established pattern. Display weight consolidated from 800 to 700 to stay within two-weight limit.

## Interaction Contract

### Tab Navigation
- Clicking an unlocked (or completed) tab switches the active phase instantly — no page reload
- Clicking a locked tab: no action, cursor `not-allowed`, tooltip visible on hover/focus
- Tab bar scrolls horizontally on viewport < 640px; active tab auto-scrolls into view on mount and on phase switch
- Keyboard: arrow keys navigate between tabs; Enter/Space activates focused tab (ARIA tablist pattern)

### Mark Complete Scroll Gate
- "Mark Complete" button is hidden on mount; appears when scroll position reaches within 100px of phase content bottom
- Exception: if content is shorter than viewport, button appears immediately
- Button disabled (opacity-50, cursor-not-allowed) after phase is already completed

### Phase 5 Quiz Gate
- Quiz submits answers; score computed client-side
- If score >= 70%: Phase 6 tab unlocks, celebration triggers, progress POST sent to server
- If score < 70%: Score displayed as "N% — need 70% to continue", retry button shown, no gate unlock

### Progress Persistence
- Authenticated users: phase completion POSTed to `/api/learning-room/:slug/progress` immediately on completion
- Anonymous users: progress held in component state for the session only; auth nudge shown after each completion
- On page load: fetch current progress for authenticated users; pre-unlock completed tabs

### Format Switching
- Switching format (when enabled) triggers immediate re-render with new content
- Format preference POSTed to server on change for authenticated users

### Phase Completion Celebration
- Confetti burst plays immediately on gate unlock
- XP toast appears 150ms after confetti start (staggered for visual clarity)
- Both dismiss after 4 seconds (matching existing XpToast pattern)

### Graph Info Panel
- Clicking a node with `has_phases = true` shows the info panel with node metadata and "Start Learning" CTA
- "Start Learning" navigates to `/learning-room/:slug`
- Nodes without `has_phases` continue to route to `/graph/:slug/learn` (ConceptPage unchanged)

## Accessibility Contract

- Tab bar: ARIA `role="tablist"`, each tab `role="tab"`, active tab `aria-selected="true"`, locked tabs `aria-disabled="true"`
- Phase content area: `role="tabpanel"` with `aria-labelledby` pointing to active tab
- Mark Complete button: `aria-label="Mark [Phase Name] complete"`
- Progress bar: `role="progressbar"`, `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="7"`
- XP toast / celebration: `role="status"`, `aria-live="polite"` (matching existing XpToast pattern)
- Locked tab tooltip: implemented as `aria-describedby` tooltip element
- Respect `prefers-reduced-motion`: confetti burst and phase-switch animations disabled; fade transitions preserved at reduced speed
- All interactive elements meet 44px minimum touch target

Source: existing accessibility patterns in `xp_toast.rs` (`role="status"`, `aria-live="polite"`) and WCAG 2.5.5 (touch targets).
