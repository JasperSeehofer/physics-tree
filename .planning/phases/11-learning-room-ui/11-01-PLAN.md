```sql
-- Add has_phases column to nodes table (per D-10 / UI-04)
ALTER TABLE nodes ADD COLUMN has_phases BOOLEAN NOT NULL DEFAULT FALSE;

-- Set has_phases=true for nodes that have entries in node_phases
UPDATE nodes SET has_phases = TRUE
WHERE id IN (SELECT DISTINCT node_id FROM node_phases);

-- Phase progress table (per D-24)
CREATE TABLE user_phase_progress (
    user_id      UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    node_id      UUID NOT NULL REFERENCES nodes(id) ON DELETE CASCADE,
    phase_number SMALLINT NOT NULL,
    completed_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    format_pref  TEXT NOT NULL DEFAULT 'reading',
    PRIMARY KEY (user_id, node_id, phase_number)
);

CREATE INDEX idx_user_phase_progress_user_node
    ON user_phase_progress(user_id, node_id);
```

2. Create `crates/server/src/handlers/learning_room.rs` with three endpoints:

**`GET /api/learning-room/:slug`** — `pub async fn get_learning_room_content`:
- Extract `State(pool)` and `Path(slug)`.
- Call `content_repo::get_node_by_slug` to get `node_id`, `title`, `branch`.
- Call `content_repo::get_phases_by_node_id` to get all `NodePhaseRow`s.
- If no node found, return 404.
- For each phase row, call `render_content_markdown(&row.content_body)` to produce HTML (SSR rendering per existing pattern).
- Return JSON response:

```rust
#[derive(Serialize)]
pub struct LearningRoomContent {
    pub node_id: String,       // UUID as string
    pub title: String,
    pub branch: String,
    pub phases: Vec<PhaseContent>,
}

#[derive(Serialize)]
pub struct PhaseContent {
    pub phase_number: i16,
    pub phase_type: String,
    pub html: String,          // pre-rendered HTML
    pub sections: Vec<String>, // h2 heading IDs (from RenderedContent.sections)
    pub simulations: Vec<String>,
}
```

**`GET /api/learning-room/:slug/progress`** — `pub async fn get_phase_progress`:
- Extract session, get `user_id`. If anonymous (None), return empty `Vec<PhaseProgressRow>`.
- Call `content_repo::get_node_by_slug` to get `node_id`.
- Call `phase_progress_repo::get_phase_progress(pool, user_id, node_id)`.
- Return JSON array of `{ phase_number, completed_at, format_pref }`.

**`POST /api/learning-room/:slug/progress`** — `pub async fn post_phase_progress`:
- Extract session, get `user_id`. If anonymous, return 401.
- Parse JSON body: `{ phase_number: i16, format_pref: String }`.
- Call `content_repo::get_node_by_slug` to get `node_id`.
- **Server-side gate (per UI-02, ROADMAP success criterion 2):** Before marking complete, enforce sequential progression:
  - If `body.phase_number == 0`, allow (first phase always accessible).
  - If `body.phase_number > 0`, call `phase_progress_repo::get_phase_progress(pool, user_id, node_id)` and check that `phase_number - 1` is in the completed set.
  - If the previous phase is NOT completed, return `403 Forbidden` with body `{"error": "Complete the previous phase first"}`.
- Only after gate passes: call `phase_progress_repo::mark_phase_complete(pool, user_id, node_id, body.phase_number, &body.format_pref)`.
- Return 200 OK.
