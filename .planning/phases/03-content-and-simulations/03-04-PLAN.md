**1. Simple Pendulum (`crates/simulation/src/mechanics/pendulum.rs`):**

The pendulum is modeled as a point mass on a rigid constraint swinging under gravity. Rapier2D does not have a built-in pendulum constraint that works well for this, so use a **custom analytical integrator** (NOT Rapier) for the pendulum — the equation of motion is `theta'' = -(g/L) * sin(theta)` which is trivially integrated via velocity Verlet.

This is an exception to using Rapier for everything. Simple pendulum is more accurately modeled analytically.

```rust
pub struct PendulumSimulation {
    // Parameters
    length: f32,       // 0.5 - 10.0 meters
    gravity: f32,      // fixed at 9.81
    initial_angle: f32, // 5 - 89 degrees
    damping: f32,      // 0.0 - 0.1 (light damping for realism)
    // State
    theta: f32,        // current angle in radians
    omega: f32,        // angular velocity
    time: f32,
    step_count: u32,
    running: bool,
    dt: f32,           // 1/60
    // Canvas
    canvas_width: f64,
    canvas_height: f64,
    // Trajectory (angle vs time for plot)
    trajectory: Vec<(f32, f32)>,
}
```

Constructor: `new(canvas_width, canvas_height)` with defaults length=2.0, initial_angle=30deg, damping=0.0.

Parameters:
- `set_length(l: f32)`: clamp 0.5..=10.0
- `set_initial_angle(deg: f32)`: clamp 5..=89
- `set_damping(d: f32)`: clamp 0.0..=0.1

`play()`: starts from current state.
`reset()`: theta = initial_angle in radians, omega = 0.

Simulation trait `step()`:
```rust
if self.running {
    // Velocity Verlet integration
    let alpha = -(self.gravity / self.length) * self.theta.sin() - self.damping * self.omega;
    self.omega += alpha * self.dt;
    self.theta += self.omega * self.dt;
    self.time += self.dt;
    self.step_count += 1;
    self.trajectory.push((self.time, self.theta));
    // NaN guard
    if self.theta.is_nan() || self.omega.is_nan() { self.reset(); }
}
```

`positions()`: convert theta to (x, y) cartesian: `x = L*sin(theta)`, `y = -L*cos(theta)` (relative to pivot).

Rendering (wasm32): Draw pivot point at top-center, rod as a line, bob as a filled circle. Draw arc showing angle. Draw trail of recent positions faded.

Presets per D-14:
- "short-fast": length=0.5, angle=45
- "long-slow": length=8.0, angle=15
- "large-swing": length=3.0, angle=80

wasm-bindgen export: `WasmPendulum` with constructor, set_length, set_initial_angle, set_damping, play, pause, reset, tick.

**2. Harmonic Oscillator (`crates/simulation/src/mechanics/harmonic.rs`):**

Spring-mass system: `F = -kx - bv` (spring force + damping). Use velocity Verlet integration (same rationale as pendulum — clean analytical model).

```rust
pub struct HarmonicSimulation {
    // Parameters
    spring_k: f32,           // 1.0 - 100.0 N/m
    mass: f32,               // 0.1 - 10.0 kg
    initial_displacement: f32, // 0.5 - 5.0 m
    damping: f32,            // 0.0 - 2.0
    // State
    x: f32,                  // displacement from equilibrium
    v: f32,                  // velocity
    time: f32,
    step_count: u32,
    running: bool,
    dt: f32,
    canvas_width: f64,
    canvas_height: f64,
    trajectory: Vec<(f32, f32)>,
}
```

Constructor: defaults k=10.0, mass=1.0, displacement=2.0, damping=0.0.

Parameters:
- `set_spring_k(k: f32)`: clamp 1.0..=100.0
- `set_mass(m: f32)`: clamp 0.1..=10.0
- `set_displacement(d: f32)`: clamp 0.5..=5.0
- `set_damping(d: f32)`: clamp 0.0..=2.0

step():
```rust
if self.running {
    let a = (-self.spring_k * self.x - self.damping * self.v) / self.mass;
    self.v += a * self.dt;
    self.x += self.v * self.dt;
    self.time += self.dt;
    self.trajectory.push((self.time, self.x));
    if self.x.is_nan() || self.v.is_nan() { self.reset(); }
}
```

Rendering: Draw spring as a zigzag line from wall (left) to block. Block is a rectangle at position x. Draw equilibrium line. Draw displacement arrow.

Presets:
- "soft-spring": k=2.0, mass=5.0, displacement=3.0
- "stiff-spring": k=80.0, mass=1.0, displacement=1.0
- "heavy-damped": k=20.0, mass=2.0, damping=1.5, displacement=4.0

wasm-bindgen export: `WasmHarmonic`.

**2. Orbital Mechanics (`crates/simulation/src/mechanics/orbital.rs`):**

2-body gravitational simulation. Use **custom integration** (not Rapier) because Rapier's gravity is uniform downward, not point-source gravitational. Use velocity Verlet with gravitational force `F = G*M*m/r^2`.

```rust
pub struct OrbitalSimulation {
    // Parameters
    central_mass: f32,    // 1e3 - 1e6 (in arbitrary units)
    orbital_mass: f32,    // 1.0 - 100.0
    initial_distance: f32, // 5.0 - 50.0
    initial_speed: f32,   // 1.0 - 50.0 (tangential)
    g_constant: f32,      // gravitational constant (tuned for visual scale)
    // State
    cx: f32, cy: f32,     // central body position (fixed at center)
    ox: f32, oy: f32,     // orbiting body position
    vx: f32, vy: f32,     // orbiting body velocity
    time: f32,
    step_count: u32,
    running: bool,
    dt: f32,
    canvas_width: f64,
    canvas_height: f64,
    trajectory: Vec<(f32, f32)>,
}
```

Constructor: central body at (0, 0), orbiting body at (initial_distance, 0), velocity (0, initial_speed).
Default: central_mass=1e4, orbital_mass=1.0, distance=20.0, speed=15.0, G=1.0.

step():
```rust
if self.running {
    let dx = self.cx - self.ox;
    let dy = self.cy - self.oy;
    let r_sq = dx * dx + dy * dy;
    let r = r_sq.sqrt();
    if r < 0.5 { self.reset(); return; } // collision guard
    let force = self.g_constant * self.central_mass * self.orbital_mass / r_sq;
    let ax = force * dx / (r * self.orbital_mass);
    let ay = force * dy / (r * self.orbital_mass);
    self.vx += ax * self.dt;
    self.vy += ay * self.dt;
    self.ox += self.vx * self.dt;
    self.oy += self.vy * self.dt;
    self.time += self.dt;
    self.trajectory.push((self.ox, self.oy));
    // Keep trajectory bounded
    if self.trajectory.len() > 2000 { self.trajectory.remove(0); }
    // NaN and escape guards
    if self.ox.is_nan() || self.oy.is_nan() || r > 1000.0 { self.reset(); }
}
```

Rendering: Draw central body as large circle (sun-amber). Draw orbiting body as smaller circle (sky-teal). Draw orbital trail from trajectory. Center the view on the central body with auto-scaling.

Presets:
- "circular": distance=20, speed=22.36 (sqrt(GM/r) for circular orbit)
- "elliptical": distance=15, speed=12.0
- "escape": distance=20, speed=40.0

Parameters:
- `set_central_mass(m: f32)`: clamp 1e3..=1e6
- `set_initial_distance(d: f32)`: clamp 5..=50
- `set_initial_speed(v: f32)`: clamp 1..=50

wasm-bindgen export: `WasmOrbital`.
