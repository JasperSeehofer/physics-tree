## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding |
| sm | 8px | Compact element spacing (`gap-2`, `mb-2`) |
| md | 16px | Default element spacing (`gap-4`, `p-4`) |
| lg | 24px | Section padding (`p-6`, `gap-6`) |
| xl | 32px | Layout gaps (`mt-8`) |
| 2xl | 48px | Major section breaks (`py-12`) |
| 3xl | 64px | Page-level spacing (`py-16`) |

Exceptions:
- Touch targets (Skip button, review queue cards): minimum 44px height per accessibility standard
- Review result card bottom-fixed zone: 24px bottom inset (`bottom-6`) matching XpToast pattern

**Source:** Established in prior phases; `stats_cards.rs` uses `p-6`, `gap-4`, `gap-2`, `mb-2` consistently.

## Color

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `void` / `bark-dark` | `#0d0f14` / `#1a1d24` | Page background (`bg-void`), card backgrounds (`bg-bark-dark`) |
| Secondary (30%) | `bark-mid` / `bark-light` | `#252932` / `#2e3340` | Dividers (`border-bark-light`), hover states, nested panels |
| Accent — primary (10%) | `leaf-green` | `#4caf7d` | Review queue CTA button, "due for review" badge, success/completion borders (mirrors XpToast success border) |
| Accent — warning | `sun-amber` | `#f4b942` | XP awarded display, star icon, overdue count badges (1-3 days mild) |
| Accent — urgency | `bloom-pink` | `#e8547a` | Severely overdue indicator (7+ days wilted badge), error states |
| Accent — info | `sky-teal` | `#3fc8d4` | Streak-related UI (streak count, freeze token display) — consistent with prior phases |
| Text primary | `petal-white` | `#f0f2f5` | All primary text |
| Text secondary | `mist` | `#8892a4` | Labels, metadata, secondary copy, wilting opacity treatment text |
| Wilting — mild | `mist` at 60% opacity | — | Graph nodes 1-3 days overdue (`opacity-60` in MiniTree) |
| Wilting — moderate | `mist` at 70% opacity + desaturate filter | — | Graph nodes 4-7 days overdue (`opacity-60` + SVG `feColorMatrix saturate=0.2`) |
| Wilting — severe | `mist` solid | `#8892a4` | Graph nodes 7+ days overdue (`opacity-40` in MiniTree, `COLORS.mist` in sigma_bridge) |
| Destructive | `bloom-pink` | `#e8547a` | Destructive actions only (none in this phase) |

**Accent reserved for:**
- `leaf-green`: "Start Review" / "Continue Review" button, review queue item borders on completion, empty-state "All caught up" heading color
- `sun-amber`: XP awarded number in review result card, overdue count badge background when 1-6 items due
- `bloom-pink`: Error copy text (`text-bloom-pink`), overdue badge when 7+ concepts severely overdue
- `sky-teal`: Streak count on review result card, review session streak contribution text
- `nebula-purple`: NOT used in this phase (reserved for mastery % card from Phase 4 pattern)

**Source:** Token definitions from `style/main.css`; accent usage patterns from `stats_cards.rs`, `xp_toast.rs`, `mastery_badge.rs`.

## Component Inventory

### New components for Phase 6

#### ReviewWidget (dashboard card)
- Location: `crates/app/src/components/dashboard/review_widget.rs`
- Layout: Same `bg-bark-dark rounded-card p-6` card pattern as stats cards
- Content: Clock/calendar SVG icon (20x20, `text-leaf-green`), label "Due for Review" in `text-sm text-mist`, count value in `text-3xl font-bold text-petal-white` (or `text-mist` em-dash when 0)
- CTA: "Start Review" button — `text-sm font-bold text-leaf-green underline hover:text-petal-white` link to `/review`
- Zero state: count shows em-dash, CTA reads "Nothing due today"
- Placement on dashboard: New row below StatsCards, full-width card (not part of the 4-column grid)

#### ReviewPage (`/review`)
- Location: `crates/app/src/pages/review.rs`
- Layout: Same `min-h-[calc(100vh-56px)] bg-void px-4 py-8 md:px-8` shell as DashboardPage. `max-w-2xl mx-auto` (narrower than dashboard's `max-w-5xl` — review is a focused single-concept flow)
- Progress indicator: `text-sm text-mist` showing "Concept N of M" at top of page
- Concept header: `text-xl font-bold text-petal-white` concept title + `text-sm text-mist` overdue indicator ("X days overdue" in `text-bloom-pink` if >= 7 days, `text-sun-amber` if 1-6 days)
- Quiz area: Reuses existing `QuizCheckpoint` component with 2-3 questions. No change to QuizCheckpoint UI
- Skip button: `text-sm text-mist underline hover:text-petal-white` — identical pattern to existing QuizCheckpoint skip button. Label: "Skip for today"
- Transitions: After submit/skip, 300ms `opacity-0` fade on quiz area, then new concept fades in at `opacity-100`
- Completion state: see "Empty/Completion State" section below

#### ReviewResultCard (inline, shown after each review quiz)
- Layout: `bg-bark-dark border border-leaf-green rounded-card px-4 py-4` (matches XpToast success border pattern)
- Content rows (each `flex items-center gap-2`):
  - Row 1: Score badge (`text-base font-bold text-petal-white`) + rating label (`text-sm text-mist`) — e.g. "85% — Good"
  - Row 2: XP row: star SVG (`text-sun-amber`) + `+N XP` in `text-base font-bold text-petal-white` + "(review)" in `text-sm text-mist`; if diminished: "(50% review)" or "(25% review)"
  - Row 3: Next review: calendar SVG (`text-mist`) + "Next review: Apr 2" in `text-sm text-mist`
  - Row 4 (conditional on streak update): flame SVG (`text-sky-teal`) + streak message in `text-sm text-petal-white`
- Auto-advance: 2 seconds after render, quiz area transitions to next concept
- Manual advance: "Next concept →" button in `text-sm font-bold text-leaf-green` (right-aligned)

#### Completion/Empty State
- Shown when review queue is exhausted
- Layout: centered flex column, `max-w-sm mx-auto text-center py-16`
- Heading: "Your garden is thriving." in `text-xl font-bold text-leaf-green`
- Body: "All caught up for today. Check back tomorrow to keep your knowledge fresh." in `text-sm text-mist`
- Visual: MiniTree SVG component (reused) showing the user's tree — no wilting since all reviewed
- Suggested concepts section heading: `text-sm font-bold text-petal-white mt-8 mb-4` "Continue Learning"
- Suggestion cards: 2-3 cards in `flex flex-col gap-3` — each `bg-bark-dark rounded-card p-4 flex items-center justify-between` with concept title (`text-sm font-bold text-petal-white`) + "Explore →" link in `text-sm text-leaf-green`

### Wilting — MiniTree
- Existing MiniTree SVG gains opacity modifiers on overdue nodes (source: RESEARCH.md Pattern 8)
- 1-3 days overdue: `opacity-75` on the SVG node element
- 4-7 days overdue: `opacity-60` + SVG `filter="url(#wilt-desaturate)"` (feColorMatrix saturate=0.2 added to SVG defs)
- 7+ days overdue: `opacity-40` + `filter="url(#wilt-desaturate)"`
- Mastery tier shape and color remain — only opacity and saturation change

### Wilting — Graph (sigma_bridge.js)
- Implemented via `applyWiltingStyle()` in `botanicalNodeReducer` after growth-stage styling (source: RESEARCH.md Pattern 7)
- 1-3 days: `hexWithAlpha(res.color, 0.6)` — faded
- 4-7 days: `hexWithAlpha(COLORS.mist, 0.7)` + `res.size *= 0.9` — desaturated with slight shrink
- 7+ days: `res.color = COLORS.mist` + `res.size *= 0.8` — fully gray/wilted

### Navbar badge
- Add "Review" link to navbar with a badge showing due count
- Badge: `text-xs font-bold text-void bg-bloom-pink rounded-full px-1.5 py-0.5` — only visible when count > 0
- Zero state: "Review" text link with no badge (`text-sm text-mist hover:text-petal-white`)
- Active state: `text-petal-white` (matches existing nav link active pattern)

## Interaction Contracts

### Review queue flow (D-05)
1. User navigates to `/review` — page fetches `GET /api/review/queue`
2. First concept shown immediately (no loading skeleton — data available before render via LocalResource)
3. Quiz renders with 2-3 questions from existing `QuizCheckpoint`
4. On quiz completion: `POST /api/review/submit` — result card slides in (300ms ease-out)
5. After 2 seconds (or user clicks "Next concept →"): result card fades out, next concept fades in
6. "Skip for today" button: `POST /api/review/skip` — immediately advances, no result card
7. After last concept: completion state renders with MiniTree and suggestions

### "Show more" button (D-06)
- Initial view: top 10 concepts by urgency
- "Show N more" button: `text-sm text-mist underline hover:text-petal-white` below the queue list
- On click: remaining items append below with `animate-fade-in` (300ms, existing keyframe)
- Button disappears after all items shown

### Dashboard widget behavior
- Widget fetches only COUNT from API (not full queue) — lightweight dashboard load
- "Start Review" link takes user to `/review`
- Widget re-renders after any review page session (stale-while-revalidate via LocalResource)

### Overdue visual on MiniTree (D-10)
- MiniTree on dashboard page receives `overdue_days: Option<f64>` per node
- Wilting applied client-side in Leptos component, not server-side
- No tooltip on wilting — visual degradation is self-evident

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (dashboard widget) | "Start Review" |
| Primary CTA (review page, mid-session) | "Next concept →" |
| Skip button | "Skip for today" |
| Show more button | "Show {N} more" |
| Empty state heading | "Your garden is thriving." |
| Empty state body | "All caught up for today. Check back tomorrow to keep your knowledge fresh." |
| Suggested concepts heading | "Continue Learning" |
| Suggestion card CTA | "Explore →" |
| Overdue indicator (1-6 days) | "{N} day{s} overdue" (in `text-sun-amber`) |
| Overdue indicator (7+ days) | "{N} days overdue" (in `text-bloom-pink`) |
| Review result — rating label | "Again" / "Hard" / "Good" / "Easy" |
| Review result — next review | "Next review: {Month Day}" |
| Review result — streak | "Streak extended to {N} days!" |
| Review result — XP diminished | "(50% review)" or "(25% review)" after XP amount |
| Review progress indicator | "Concept {N} of {M}" |
| Navbar badge zero state | "Review" (no badge) |
| Navbar badge non-zero | "Review" + badge showing count |
| Dashboard widget zero state count | em-dash (—) |
| Dashboard widget zero state CTA | "Nothing due today" (no link) |
| Error state — queue fetch failed | "Couldn't load your review queue. Refresh to try again." |
| Error state — submit failed | "Couldn't save your review. Try again." |

**Tone:** Warm and botanical. "Thriving," "fresh," "garden" vocabulary consistent with the botanical metaphor established in Phase 1. No urgent or alarming language even for overdue states — wilting is a gentle reminder, not a punishment.

**Destructive actions in this phase:** None. Skip defers — it does not delete or reset any data. No confirmation required.
