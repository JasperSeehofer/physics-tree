       **get_review_queue(pool, user_id) -> Result<Vec<ReviewQueueItem>, sqlx::Error>:**
       Query: SELECT from progress JOIN nodes WHERE user_id=$1 AND next_review IS NOT NULL AND next_review <= NOW(). Order by days_overdue DESC, mastery_level DESC. Use `EXTRACT(EPOCH FROM (NOW() - p.next_review)) / 86400.0 AS days_overdue`. Return all items (D-06 soft cap handled client-side).

       **submit_review(pool, user_id, node_id, score_pct) -> Result<SubmitReviewResult, sqlx::Error>:**
       Steps:
       a. Read current FSRS state from progress row (fsrs_stability, fsrs_difficulty, fsrs_elapsed_days, fsrs_scheduled_days, fsrs_reps, fsrs_lapses, fsrs_state, last_reviewed, next_review)
       b. Build FsrsCard from DB columns
       c. Call fsrs_logic::schedule_review(card, score_pct, Utc::now()) to get next card state
       d. Count review events this week: `SELECT COUNT(*) FROM xp_events WHERE user_id=$1 AND node_id=$2 AND is_review=TRUE AND occurred_at >= NOW() - INTERVAL '7 days'` (rolling 7 days per Pitfall 2)
       e. Compute XP: fetch depth_tier from nodes, call xp_logic::compute_xp(depth_tier, score_pct), multiply by fsrs_logic::review_xp_multiplier(review_count_this_week)
       f. Update progress row: SET fsrs_stability, fsrs_difficulty, fsrs_elapsed_days, fsrs_scheduled_days, fsrs_reps, fsrs_lapses, fsrs_state, last_reviewed=NOW(), next_review=new_card.due
       g. Insert xp_events row with is_review=TRUE (the idempotency check must filter by is_review — a review on the same day as an initial quiz is allowed per Pitfall 5)
       h. Update mastery_level and xp_earned on progress (add xp_amount)
       i. Return SubmitReviewResult { xp_awarded, rating_label, next_review_date, new_mastery_level }

       **SubmitReviewResult struct:**
       ```rust
       #[derive(Debug, Serialize)]
       pub struct SubmitReviewResult {

           pub new_mastery_level: i32,
       }
       ```

       **skip_review(pool, user_id, node_id) -> Result<(), sqlx::Error>:**
       UPDATE progress SET next_review = NOW() + INTERVAL '24 hours' WHERE user_id=$1 AND node_id=$2. Do NOT modify any FSRS columns (stability, difficulty, reps, lapses, state) per Pitfall 6.

       SELECT COUNT(*) FROM progress WHERE user_id=$1 AND next_review IS NOT NULL AND next_review <= NOW(). For dashboard widget (lightweight).

Use edges table (from_node is prereq, to_node is dependent). ORDER BY RANDOM() LIMIT $2. Per D-13.
