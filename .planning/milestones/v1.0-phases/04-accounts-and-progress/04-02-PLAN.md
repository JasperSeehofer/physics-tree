**LoginForm component** (`crates/app/src/components/auth/login_form.rs`):

Create a `#[component] pub fn LoginForm() -> impl IntoView` that renders:
- Email input: `<input type="email" name="email" />` with `bg-bark-mid border border-bark-light rounded px-4 py-3 text-petal-white placeholder-mist text-base focus:border-leaf-green focus:outline-none w-full`
- Password input: same styling, `type="password"`
- Labels above each input: `<label class="text-sm font-bold text-petal-white mb-1 block">` with explicit `for` attribute matching input `id` (per accessibility contract)
- Error display: `<p class="text-sm text-bloom-pink mt-1" role="alert">` below field, shown conditionally
- Submit button: `<button type="submit" class="w-full py-3 rounded-lg bg-leaf-green text-void font-bold text-sm hover:brightness-110 disabled:opacity-60">"Log In"</button>`
- Loading state: `RwSignal<bool>` — when true, button text becomes "Logging in..." and button is disabled

State management with RwSignals:
- `email: RwSignal<String>`, `password: RwSignal<String>`
- `error: RwSignal<Option<String>>` for server errors
- `submitting: RwSignal<bool>` for loading state
- `email_error: RwSignal<Option<String>>`, `password_error: RwSignal<Option<String>>` for field-level errors

Validation on blur (per interaction contract):
- Email: non-empty and contains '@'. Error: "Enter a valid email address."
- Password: non-empty. Error: no field-level error for password on login (server validates)

On submit: use `gloo-net` (already in app deps for wasm32) to POST to `/api/auth/login` with `{ email, password }`. Guard behind `#[cfg(target_arch = "wasm32")]`. On success (200), navigate to `/dashboard` using `leptos_router::hooks::use_navigate()`. On 401, set error to "Email or password is incorrect. Try again or reset your password." On other errors, set generic error.
