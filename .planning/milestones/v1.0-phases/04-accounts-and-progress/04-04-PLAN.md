**Graph explorer bottom sheet** (`crates/app/src/components/graph/panel.rs` and `crates/app/src/pages/graph_explorer.rs`):

Per D-18: On screens <= 768px (below `lg:` breakpoint), the node detail panel renders as a bottom sheet overlay instead of a right sidebar.

Modify the panel component or its parent layout to use responsive Tailwind classes:

1. **Desktop (lg+):** Keep existing right sidebar behavior — panel is a column next to the graph canvas. Use `lg:relative lg:w-[360px] lg:h-full` or similar to maintain current layout.

2. **Mobile/tablet (below lg):** Panel becomes a bottom sheet:
   - Container: `fixed bottom-0 left-0 right-0 bg-bark-dark rounded-t-2xl border-t border-bark-light z-50 max-h-[60vh] overflow-y-auto`
   - Add a drag handle at top: `<div class="w-12 h-1 bg-bark-light rounded mx-auto mt-3 mb-2"></div>`
   - Panel content stays the same — just the positioning changes.

3. **Graph explorer page layout** (`graph_explorer.rs`): Adjust the flex/grid layout so the graph canvas fills the full width on mobile (no reserved sidebar space). Use `lg:grid lg:grid-cols-[1fr_360px]` for desktop, plain `w-full` for mobile. The graph canvas should be `w-full h-full` at all sizes.

Implementation approach: Use Tailwind responsive classes directly. The panel component already controls its visibility via `panel_open` signal. Add the responsive positioning classes:

```rust
// The panel container switches between sidebar (lg+) and bottom sheet (below lg)
<div class={move || {
    let base = "bg-bark-dark border-bark-light overflow-y-auto";
    if panel_open.get() {
        format!("{base} fixed bottom-0 left-0 right-0 rounded-t-2xl border-t z-50 max-h-[60vh] lg:relative lg:bottom-auto lg:left-auto lg:right-auto lg:rounded-none lg:border-t-0 lg:border-l lg:max-h-full lg:w-[360px]")
    } else {
        "hidden".to_string()
    }
}}>
```

Or use simpler approach: two separate elements with `lg:hidden` / `hidden lg:block` to render different layouts. Choose whichever is cleaner given the current panel.rs structure.

**Content TOC overlay** (`crates/app/src/components/content/toc.rs` and `crates/app/src/pages/concept.rs`):

Per D-19: On screens <= 768px, TOC sidebar is hidden by default and revealed as an overlay via a toggle button.

1. Add a `toc_open: RwSignal<bool>` signal (default false).

2. **Desktop (lg+):** Keep existing sticky sidebar. Use `hidden lg:block lg:sticky lg:top-14` (top offset = navbar height).

3. **Mobile/tablet (below lg):** TOC is hidden by default. Add a toggle button in the concept page header:
   - Button: `<button class="lg:hidden w-8 h-8 flex items-center justify-center text-mist hover:text-petal-white" aria-label="Table of contents" on:click=move |_| toc_open.update(|v| *v = !*v)>` — use a list/menu inline SVG icon.
   - Backdrop: `<div class="fixed inset-0 z-40 bg-void/80 lg:hidden" on:click=move |_| toc_open.set(false)>` (shown when toc_open is true)
   - TOC panel: `<div class="fixed top-0 left-0 h-full w-64 bg-bark-dark border-r border-bark-light z-50 overflow-y-auto p-6 lg:hidden">` (shown when toc_open is true)

4. **Concept page layout** (`concept.rs`): Adjust the grid to give content full width on mobile. Use `lg:grid lg:grid-cols-[240px_1fr]` for desktop, `w-full` for mobile. Content column: `w-full px-4 lg:px-0`. Simulations: `w-full` at all sizes (per D-19: "Simulations stay full-width").

5. Close TOC overlay when a section link is clicked (add `on:click=move |_| toc_open.set(false)` to section links in the TOC).

**General responsive checks:** Ensure no fixed pixel widths on the main content area. The landing page, graph explorer, and concept pages should all work at 640px minimum width. Look for any `w-[NNNpx]` or `min-w-[NNNpx]` classes that might cause overflow and replace with responsive alternatives.
