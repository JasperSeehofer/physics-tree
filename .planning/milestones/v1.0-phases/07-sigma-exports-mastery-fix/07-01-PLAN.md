<objective>
Close the sigma_entry.js export gap and fix MasteryBadge per-concept XP.

Gap 1: sigma_entry.js bundles sigma_bridge.js but only exports 6 functions to window.__sigma_bridge.
The functions updateUserProgress and updateOverdueMap exist in sigma_bridge.js but are missing
from the sigma_entry.js re-export object. When an authenticated user loads /graph, the Rust WASM
calls js::update_user_progress() which calls window.__sigma_bridge.updateUserProgress but it
doesn't exist, causing a WASM panic via the .expect("updateUserProgress not found") call.

Gap 2: MasteryBadge on concept pages uses mastery_xp which is initialized to 0 (hidden) and
only updated from award-xp response's new_total_xp field. But new_total_xp is the user's
aggregate total XP across all concepts — not the per-concept mastery level. Badge shows
incorrect (aggregate) XP. Also, the badge only appears after quiz completion, not on page load.

Fix: Export the missing functions in sigma_entry.js. Add concept_mastery_xp field to
AwardXpResponse (already computed as new_concept_xp in award_xp handler). Add
GET /api/progress/node/:node_id endpoint to fetch per-concept mastery on load.
Wire concept.rs to fetch mastery on load and set from concept_mastery_xp on award.
</objective>
