# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Phase scope: Rust workspace setup, database schema, CI pipeline, and the app shell — a branded landing page with the PhysicsTree wordmark, botanical placeholder illustration, and a health-check status indicator. No graph, no content, no auth.

---

| Property | Value |

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline text padding |
| sm | 8px | Compact element spacing (badge padding, icon-label gap) |
| md | 16px | Default element spacing (card padding, form field gap) |
| lg | 24px | Section padding (between header and body) |
| xl | 32px | Layout gaps (between major landing page sections) |
| 2xl | 48px | Major section breaks (hero vertical rhythm) |
| 3xl | 64px | Page-level spacing (hero top/bottom padding) |

Exceptions:
- Health-check indicator pill: 8px vertical / 16px horizontal padding (sm/md) — intentionally compact, not a button
- Wordmark SVG: 40×40px intrinsic size — not a spacing token, but a fixed asset dimension

**Source:** 8-point scale standard. No upstream override.

## Typography

| Role | Size | Weight | Line Height | Tailwind token |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 (regular) | 1.6 | `text-base font-normal` |
| Label | 14px | 400 (regular) | 1.4 | `text-sm font-normal` |
| Heading | 24px | 700 (bold) | 1.3 | `text-2xl font-bold` |

Font family declaration in `@theme {}`:
```css
--font-sans: "Nunito", system-ui, sans-serif;
--font-display: "Nunito", system-ui, sans-serif;
```

Notes:
- Heading size (24px) is used for section labels and the tagline.
- Label size (14px) is used for the health-check pill text and metadata. At 14px, `font-normal` is visually distinct from 16px body without a weight change.
- System weights are 400 (regular) and 700 (bold) only.
- Nunito must be loaded at weights 400 and 700 from `public/fonts/`.

**Source:** Nunito selection from RESEARCH.md discretion recommendation. Sizes chosen to establish a clear visual hierarchy for the landing page.

## Color

All colors are dark-only. No light mode. The `.dark` class is applied to `<html>` unconditionally.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--color-void` | `#0d0f14` | Page background (`<body>`, full-bleed sections) |
| Secondary (30%) | `--color-bark-dark` + `--color-bark-mid` | `#1a1d24` / `#252932` | Primary surface / elevated surfaces (cards, panels, status pill background) |
| Accent primary (10%) | `--color-leaf-green` | `#4caf7d` | Reserved for: wordmark "Tree" text highlight, health-check PASS state, primary CTA button background |
| Accent secondary | `--color-sky-teal` | `#3fc8d4` | Reserved for: SVG highlight leaf in wordmark illustration |
| Destructive | `--color-bloom-pink` | `#e8547a` | Reserved for: health-check FAIL state indicator only (no destructive actions in Phase 1 UI) |

Additional palette tokens declared in `@theme {}` for use by future phases (do NOT use in Phase 1 components beyond what is listed above):

| Token | Hex | Future use |
|-------|-----|------------|
| `--color-bark-light` | `#2e3340` | Borders, dividers |
| `--color-sun-amber` | `#f4b942` | XP, streaks, gold mastery (Phase 5) |
| `--color-nebula-purple` | `#8b5cf6` | Theorem nodes (Phase 2) |
| `--color-petal-white` | `#f0f2f5` | Primary text |
| `--color-mist` | `#8892a4` | Secondary text, metadata |

Accent reserved for (Phase 1 only):
1. The word "Tree" in the PhysicsTree wordmark (leaf-green)
2. Health-check PASS pill background tint (`leaf-green/20`) and text (`leaf-green`)
3. SVG illustration highlight leaf (`sky-teal` at 60% opacity)
4. Health-check FAIL pill background tint (`bloom-pink/20`) and text (`bloom-pink`)

Accent is NOT used on: body text, card borders, generic hover states, page background.

**Source:** Hex values from RESEARCH.md Pattern 2 (Tailwind v4 `@theme {}` example). Role assignments derived from CONTEXT.md visual direction and DSGN-01 requirement.

- No sidebar. No grid. No columns.

| Element | Copy |
|---------|------|
| Page title (`<title>`) | `PhysicsTree — Explore the physics universe` |
| Hero tagline | `Explore the interconnected landscape of physics` |
| Health indicator — PASS | `System operational` |
| Health indicator — FAIL | `System unavailable` |

| Empty state body | Not applicable — Phase 1 has no data-driven views |

**Source:** CONTEXT.md ("The landing page should feel like arriving at something special, not a generic 'under construction' page"). Copy drafted to match Kurzgesagt's matter-of-fact playful tone.

## Interaction Contract

Phase 1 interactions are minimal — this is a static landing page with one async data fetch.

| Interaction | Behavior |
|-------------|----------|
| Page load | Health check fetch fires immediately via Leptos `Resource`. Pill shows "Checking system…" (mist color) until resolved. |
| Health PASS | Pill transitions to leaf-green tint. No animation — instant render on data resolve. |
| Health FAIL | Pill transitions to bloom-pink tint. No animation — instant render on data resolve. |
| Header wordmark click | No-op in Phase 1 (links to `/`, same page). |
| All other elements | Static, non-interactive. |

No hover states, focus rings, or keyboard interactions defined for Phase 1 — there are no interactive elements beyond what the browser provides by default. The health indicator is display-only.

**Accessibility baseline (applies to all phases from this point forward):**
- `<html lang="en">` declared
- Color contrast ratio: all text on dark backgrounds must meet WCAG AA (4.5:1 for body text, 3:1 for large text). `#f0f2f5` on `#0d0f14` = ~14:1 — passes.
- `<title>` set per page
- SVG wordmark includes `aria-hidden="true"` (decorative; wordmark text is in adjacent `<h1>`)

## Tailwind v4 Token Reference

The canonical token file is `style/main.css`. Excerpt for Phase 1 (complete file shown in RESEARCH.md Pattern 2):

```css
@import "tailwindcss";
@source "../crates/app/src/**/*.rs";
@custom-variant dark (&:where(.dark, .dark *));

@theme {
  --color-void: #0d0f14;
  --color-bark-dark: #1a1d24;
  --color-bark-mid: #252932;
  --color-bark-light: #2e3340;
  --color-leaf-green: #4caf7d;
  --color-bloom-pink: #e8547a;
  --color-sun-amber: #f4b942;
  --color-sky-teal: #3fc8d4;
  --color-nebula-purple: #8b5cf6;
  --color-petal-white: #f0f2f5;
  --color-mist: #8892a4;
  --font-sans: "Nunito", system-ui, sans-serif;
  --font-display: "Nunito", system-ui, sans-serif;
  --radius-node: 0.75rem;
  --radius-card: 0.5rem;
}

@layer base {
  html { @apply dark; }
  body {
    background-color: var(--color-void);
    color: var(--color-petal-white);
    font-family: var(--font-sans);
  }
}
```

Tailwind class naming convention in Leptos RSX: use full literal class strings only. Do NOT construct class names dynamically (e.g., `format!("bg-{}", color)`) — Tailwind's scanner cannot detect dynamic strings. Use conditional expressions that produce complete class strings (e.g., `if ok { "bg-leaf-green/20 text-leaf-green" } else { "bg-bloom-pink/20 text-bloom-pink" }`).
