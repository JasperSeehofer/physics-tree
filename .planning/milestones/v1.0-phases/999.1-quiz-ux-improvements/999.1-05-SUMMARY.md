## Decisions Made

- `Result<AwardXpResponse, bool>` instead of `Option<AwardXpResponse>` — `bool` error type cleanly encodes the single relevant distinction: auth failure vs any other failure. Using a custom enum would be over-engineering for a local async helper.
- Inline nudge placement (after quiz checkpoints, before NextConceptNav) — puts the message where the user's attention is after completing the quiz, without interrupting the learning flow with a toast or modal.
- `login_nudge.set(false)` in the data-fetch Effect (which subscribes to `slug()`) — same Effect already resets all other state on navigation, so this is the canonical reset point.
