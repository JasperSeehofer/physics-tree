## What Was Built

### Task 1: LaTeX Rendering Infrastructure

**content.rs:**
- Imported `extract_latex_placeholders` from `markdown_renderer`
- Applied to all 4 quiz string fields: `question`, `hint`, `explanation`, and each `option.text`
- Also applied to matching pair `(left, right)` strings

**multiple_choice.rs, formula_input.rs, matching.rs:**
- Question text changed from `{question_text}` to `inner_html=question_text`
- Hint and explanation text changed to `inner_html` directives
- Option/pair labels use `inner_html` on the text span
- Added `#[cfg(target_arch = "wasm32")] Effect::new` subscribed to state signal — calls `renderAllPlaceholders()` after each state change so LaTeX in feedback text gets rendered

**concept.rs (D-07 fix):**
- Converted one-shot `#[cfg(target_arch = "wasm32")] { spawn_local }` block to a reactive `Effect::new(move |_| { ... })` that reads `slug()` — causes re-execution when the route slug changes (client-side navigation)
- Resets content/quiz/checkpoint state at the start of each re-run to avoid stale data

### Task 2: Hint Tracking Through Callback Chain

**multiple_choice.rs, formula_input.rs:**
- `hint_shown: RwSignal<bool>` added; set to `true` when hint displayed
- `on_correct: Callback<bool>` — passes `hint_shown.get()` on correct answer

**matching.rs:**
- `hint_shown: RwSignal<bool>` — set to `true` when check result is wrong (user sees hint in feedback)
- `on_correct: Callback<bool>` — passes `hint_shown.get()`

**checkpoint.rs:**
- `on_answered: Callback<(bool, bool)>` — tuple `(correct, hint_used)` propagated from inner components
- Skip button passes `(false, false)`

**concept.rs:**
- `checkpoint_passed: RwSignal<Vec<Option<(bool, bool)>>>` — tuple per checkpoint
- `correct_count` uses `matches!(p, Some((true, _)))`

**review.rs:**
- `on_correct` closures accept `_hint_used: bool` and ignore it — reviews go through separate endpoint without hint penalty

### Task 3: End-to-End Hint Penalty XP Flow

**concept.rs:**
- `AwardXpRequest` gains `hints_used: bool`
- `AwardXpResponse` gains `hint_penalty: bool`
- `any_hints_used` aggregated from `checkpoint_passed`
- `post_award_xp` passes `any_hints_used`
- `XpAwardData` created with `hint_penalty: response.hint_penalty`

**progress.rs (server):**
- `AwardXpRequest.hints_used: bool` added
- `AwardXpResponse.hint_penalty: bool` added
- `compute_xp(..., req.hints_used)` — 50% penalty applied server-side
- `is_perfect` suppressed when `hints_used` (no 1.5x bonus after using hint)
- Both response paths (below-threshold and success) include `hint_penalty`

**xp_toast.rs:**
- `XpAwardData.hint_penalty: bool` added
- Toast renders `<span class="text-sm text-sky-teal">"hint penalty applied"</span>` when active

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - D-07 architectural fix] Reactive slug-based Effect for data fetch**

- **Found during:** Task 1, Part C verification
- **Issue:** Leptos Router does NOT recreate the ConceptPage component on client-side navigation between routes matching the same pattern (`/graph/:slug/learn`). The one-shot `spawn_local` block only ran once at component mount, so content/quiz didn't re-fetch when navigating between concept pages.
- **Fix:** Converted the one-shot `#[cfg(target_arch = "wasm32")] {}` block to a `#[cfg(target_arch = "wasm32")] Effect::new(move |_| { ... })` that subscribes to `slug()`. Leptos tracks this dependency and re-runs the Effect when the slug param changes (navigation). State is reset at the start of each re-run.
- **Files modified:** `crates/app/src/pages/concept.rs`
- **Commit:** ea651e5
