The problem: `post_award_xp` returns `None` on any HTTP error (including 401 unauthorized). The XP Effect does nothing when it gets `None`, so users who aren't logged in see no toast and no feedback — they don't know they're missing XP.

**Step 1: Change post_award_xp return type to distinguish auth failure**

Change `post_award_xp` to return `Result<AwardXpResponse, bool>` where:
- `Ok(response)` = success
- `Err(true)` = auth failure (401)
- `Err(false)` = other error (network, server error, etc.)

```rust
#[cfg(target_arch = "wasm32")]
async fn post_award_xp(node_id: String, score_pct: u32, hints_used: bool) -> Result<AwardXpResponse, bool> {
    let body = serde_json::to_string(&AwardXpRequest { node_id, score_pct, hints_used })
        .map_err(|_| false)?;
    let resp = gloo_net::http::Request::post("/api/progress/award-xp")
        .header("Content-Type", "application/json")
        .body(body)
        .map_err(|_| false)?
        .send()
        .await
        .map_err(|_| false)?;

    if resp.status() == 401 {
        return Err(true); // Auth failure
    }
    if !resp.ok() {
        return Err(false); // Other error
    }
    resp.json::<AwardXpResponse>().await.map_err(|_| false)
}
```
