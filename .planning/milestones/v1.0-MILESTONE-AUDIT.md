### Unsatisfied / Partial Requirements

**GRAPH-05: Personal knowledge tree grows visually** — UNSATISFIED
- `sigma_bridge.js` correctly implements `updateUserProgress` with botanical growth stages
- `sigma_entry.js` does NOT export `updateUserProgress` to `window.__sigma_bridge`
- `canvas.rs` calls `js_sys::Reflect::get("updateUserProgress")` which returns `undefined` at runtime
- **Result:** WASM panic crashes the graph page for authenticated users; botanical growth never renders
- **Fix:** Add `updateUserProgress` and `updateOverdueMap` to imports and exports in `sigma_entry.js`, rebuild `sigma_bundle.js`

**GAME-03: Mastery levels tied to plant growth visual** — PARTIAL
- Backend mastery logic correct (`xp_to_mastery_tier` thresholds, `compute_xp` depth-tier scaling)
- MasteryBadge on concept page shows wrong tier: `concept.rs:271` uses `response.new_total_xp` (aggregate across ALL concepts) instead of per-concept XP
- Graph-side mastery rendering blocked by the same `sigma_entry.js` gap as GRAPH-05
- **Fix:** Use `response.mastery_tier` string or expose per-concept XP in `AwardXpResponse`
